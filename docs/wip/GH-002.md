  Ubuntu system.
- Measured the server-only idle baseline before endpoint provisioning.

  local audio paths needed by the current Planpoint.

# Confirmation

## Automated Checks

- Command: Inspect the operating system with `/etc/os-release`, `uname`,
  `systemctl`, and the current APT package index.
- Result: Ubuntu 24.04.4 LTS runs kernel `6.8.0-136-generic`; no failed systemd
  units or currently indexed package upgrades were reported.
- Command: Inspect hardware with DMI sysfs, `lscpu`, `lshw`, `free`, `lsblk`,
  `lspci -nnk`, `lsusb`, ALSA procfs, and DRM sysfs.
- Result: The inventory below includes 4 GiB installed memory, 3.8 GiB usable
  memory, and Intel ICH8 UHCI/EHCI USB 1.1/2.0 controllers. The expected
  `radeon`, `sky2`, `snd_hda_intel`, `applesmc`, and `b43` kernel modules were
  loaded.
- Command: Read the internal panel's EDID size and DRM modes plus the active
  framebuffer.
- Result: The connected 52 × 32 cm panel exposes native 1920 × 1200 output
  through a 32-bit `radeondrmfb` framebuffer, identifying the 24-inch model.
- Command: Sample `vmstat`, procfs memory and load, hwmon temperatures, and
  Apple SMC fan readings 16 times at 60-second intervals.
- Result: CPU remained 97–100% idle, about 3.4 GiB memory remained available,
  and swap remained unused. CPU cores measured 54–58°C and the Radeon measured
  75–77°C. Optical-drive, hard-drive, and CPU fans respectively held at
  985–997, 1207–1211, and 1197–1201 RPM.
- Command: Inspect `/sys/class/net`, `networkctl`, the boot kernel journal, and
  `apt-cache policy firmware-b43-installer`.
- Result: Ethernet is up at 1 Gbit/s full duplex. The BCM4321 is detected, but
  `b43/ucode11.fw` is absent, so no wireless interface exists. Ubuntu provides
  `firmware-b43-installer` from Noble multiverse for GH-003.
- Command: Inspect ALSA devices and controls with `/proc/asound`, `aplay`, and
  `amixer`, then run separate temporary `speaker-test` streams with the speaker
  and headphone controls isolated.
- Result: ALSA accepted both streams through ALC889A analog device `0,0`.
  `Master` was initially muted at 0%, explaining the earlier silent test. The
  original mixer state was restored after each stream.
- Command: Enumerate USB devices with `lsusb`.
- Result: Built-in Bluetooth, iSight, and infrared devices, an Apple keyboard,
  and a wireless input receiver were detected. No USB device is required for
  this Planpoint.
- Command: Perform one password-authorized reboot while polling key-based SSH,
  then recheck Ethernet, services, drivers, and device nodes.
- Result: The iMac went offline and returned to SSH without local intervention.
  At 38 seconds uptime Ethernet was up at 1 Gbit/s, SSH was active, required
  drivers were loaded, and no systemd units had failed.
- Command: After the unattended reboot, inspect ALSA enumeration, device-node
  ownership, account groups, and local login sessions.
- Result: Audio and graphics devices exist, but the SSH-only `imac` account is
  not in the `audio`, `video`, or `render` groups and has no local-session ACL.
  `aplay` therefore reports no usable sound card until GH-003 provides an
  intentional local session or service-level device access.

## Manual Testing

  runs.
- Action: Listen for the 440 Hz left-then-right test with the headphone control
  muted.
- Expected result: The tone is audible only from the iMac speakers.
- Result: Confirmed by the user.
- Start: Observe the iMac after at least fifteen minutes at server idle.
- Action: Compare the fan sound with the preliminary quiet baseline.
- Expected result: Fan noise remains steady and acceptable for the room.
- Result: Awaiting user confirmation.
- Start: After GH-003 enables Wi-Fi, move the iMac into place and connect its
  rear analog output to the Sonos line-in.
- Action: Repeat the isolated headphone-output test and select the Sonos
  line-in.
- Expected result: The tone is audible through the Sonos and not the iMac
  speakers.
- Result: Deferred to GH-003 because the physical connection is not yet
  available.
